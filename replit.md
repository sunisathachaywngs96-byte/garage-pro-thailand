# Workspace

## Overview

pnpm workspace monorepo using TypeScript. Each package manages its own dependencies.

## Stack

- **Monorepo tool**: pnpm workspaces
- **Node.js version**: 24
- **Package manager**: pnpm
- **TypeScript version**: 5.9
- **API framework**: Express 5
- **Database (app)**: PostgreSQL + Drizzle ORM
- **Database (SaaS)**: Cloud Firestore (multi-tenant)
- **Validation**: Zod (`zod/v4`), `drizzle-zod`
- **API codegen**: Orval (from OpenAPI spec)
- **Build**: esbuild (CJS bundle)

## Key Commands

- `pnpm run typecheck` — full typecheck across all packages
- `pnpm run typecheck:libs` — typecheck composite libs only (db, api-zod, api-client-react, garage-schema)
- `pnpm run build` — typecheck + build all packages
- `pnpm --filter @workspace/api-spec run codegen` — regenerate API hooks and Zod schemas from OpenAPI spec
- `pnpm --filter @workspace/db run push` — push DB schema changes (dev only)
- `pnpm --filter @workspace/api-server run dev` — run API server locally
- `firebase deploy --only firestore:rules` — deploy Firestore security rules
- `firebase deploy --only firestore:indexes` — deploy Firestore indexes
- `GOOGLE_APPLICATION_CREDENTIALS=./service-account.json TENANT_ID=demo-garage-01 pnpm --filter @workspace/scripts run seed:firestore` — seed Firestore dev data

## Libraries (`lib/`)

### `@workspace/db`
PostgreSQL schema + Drizzle ORM. Tables: customers, vehicles, services, technicians, appointments.

### `@workspace/api-spec`
OpenAPI spec source. Run codegen to update client hooks and Zod schemas.

### `@workspace/api-zod`
Zod schemas generated from OpenAPI spec (exports from `./generated/api` only — no `schemas` subpath to avoid duplicate export collision).

### `@workspace/api-client-react`
React Query hooks generated by Orval. All hooks live in `src/generated/api.ts`.

### `@workspace/garage-schema`
**Firestore schema for Thai garage SaaS multi-tenant architecture.**
- Path: `lib/garage-schema/`
- Collections (all subcollections under `/tenants/{tenantId}/`):
  - `users` — staff with roles and permissions
  - `customers` — individual or business clients
  - `vehicles` — vehicles linked to customers
  - `jobs` — work orders with services, parts, and pricing
  - `payments` — payment records per job (multi-payment support)
  - `stock_items` — inventory with movement subcollection
- Exports: TypeScript types, `ROLE_PERMISSIONS` matrix, `col`/`docPath` path helpers
- No runtime dependencies — uses a local `Timestamp` interface compatible with `firebase/firestore`

## Firestore Files (root)

| File | Purpose |
|------|---------|
| `firestore.rules` | Security rules — multi-tenant isolation + RBAC (owner/manager/receptionist/technician/viewer) |
| `firestore.indexes.json` | Composite indexes for jobs, customers, vehicles, payments, stock |
| `storage.rules` | Firebase Storage rules — per-path upload restrictions by role |
| `firebase.json` | Firebase project config — Firestore, Functions, Storage, Emulator Suite |

## Auth Architecture (Firestore SaaS)

Custom claims on Firebase Auth tokens (set by Cloud Function):
```json
{ "tenantId": "demo-garage-01", "role": "manager" }
```
Rules read `request.auth.token.tenantId` and `request.auth.token.role` — no extra document reads per rule evaluation.

## Role Hierarchy

```
viewer < technician < receptionist < manager < owner
```

Each role's permissions are in `lib/garage-schema/src/roles.ts` → `ROLE_PERMISSIONS`.

## DB Schema (lib/db/src/schema/) — PostgreSQL
- `customers.ts` — customer records
- `vehicles.ts` — vehicle registry linked to customers
- `services.ts` — service catalog with pricing
- `technicians.ts` — staff roster with availability
- `appointments.ts` — bookings linking all entities

## Notes
- `lib/api-zod/src/index.ts` exports only from `./generated/api` (not `./generated/types`) to avoid Orval duplicate export collision
- Seed data (PostgreSQL): 6 customers, 7 vehicles, 8 services, 4 technicians, 14 appointments
- Seed data (Firestore): `scripts/src/seed-firestore.ts` — 1 demo tenant, 4 users, 3 customers, 2 vehicles, 2 jobs, 1 payment, 3 stock items
- `lib/garage-schema` uses a local `Timestamp` interface (no `firebase` package dep) — compatible with real Firestore `Timestamp` at runtime
